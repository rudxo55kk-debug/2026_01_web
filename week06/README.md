# week06 학습 정리

- 작성일: 2026-04-08

## 1) 클래스 기반 도형 실습 (RectangleDemo)
- 클래스의 생성자와 기본 생성자를 사용해 객체를 생성하는 방법을 연습했다.
- 좌표와 크기 값의 범위를 제한(clamp)해 안정적으로 객체를 다루는 방법을 확인했다.
- static 변수를 통해 전체 객체 수를 관리하는 방식을 학습했다.

## 2) 인터페이스 기반 도형 설계 (ShapeDemo)
- Shape 인터페이스를 기준으로 Rectangle, Triangle, Circle을 구현했다.
- 같은 메서드(move, translate, rotate, printInfo)를 도형별로 다르게 구현해 다형성을 확인했다.
- 값이 주어지지 않을 때 무작위 좌표/크기를 생성하는 기본 생성자 패턴을 적용했다.

## 3) 인터페이스 확장 버전 (InterfaceDemo / ShapeDemoInterfaceDemo)
- inter- 접두어(InterShape, InterRectangle, InterTriangle, InterCircle) 규칙으로 별도 구현을 만들었다.
- 기존 구조와 같은 기능을 다른 이름 규칙으로 재구성하여 설계 유연성을 연습했다.

## 4) 익명 클래스 실습 (ShapeAnonymous)
- Shape 타입을 이용해 익명 클래스로 객체를 직접 생성했다.
- 이름 없는 구현체에서도 메서드 오버라이딩과 상태 관리가 가능함을 확인했다.
- 간단한 출력 예제를 통해 익명 클래스의 사용 목적(일회성 구현)에 익숙해졌다.

## 5) 오늘 학습 핵심
- 객체지향 설계에서 클래스, 인터페이스, 익명 클래스의 차이를 비교했다.
- 공통 동작은 인터페이스로 추상화하고, 구체 동작은 구현체에서 분리하는 구조를 연습했다.
- 생성자, static 카운트, 무작위 초기화, 다형성 호출 흐름을 종합적으로 복습했다.
